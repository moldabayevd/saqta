# Архитектура

## Общий принцип

Проект следует Unix-way — маленькие инструменты, каждый делает одну вещь, связываются через файловую систему.

```
┌─────────────────┐    ⌘⇧R     ┌─────────────────┐
│  QuickRecorder  │──────────▶│  ~/Recordings/  │
│   (menu bar)    │  MP4+audio │   new file.mp4  │
└─────────────────┘            └────────┬────────┘
                                        │
                               fswatch detects
                                        │
                                        ▼
                         ┌──────────────────────────┐
                         │   auto-transcribe.sh     │
                         │   (LaunchAgent daemon)   │
                         └────────────┬─────────────┘
                                      │
        ┌─────────────────────────────┼─────────────────────────────┐
        ▼                             ▼                             ▼
 ┌─────────────┐              ┌──────────────┐             ┌───────────────┐
 │   ffmpeg    │              │ whisper.cpp  │             │  terminal-    │
 │ extract WAV │  ───────▶   │  + Metal     │  ────────▶ │  notifier     │
 │ 16kHz mono  │              │  transcribe  │             │  (уведомления)│
 └─────────────┘              └──────┬───────┘             └───────────────┘
                                     │
                                     ▼
                    ┌─────────────────────────────┐
                    │  meeting_YYYY-MM-DD_HH-MM/  │
                    │  ├── video.mp4              │
                    │  ├── transcript.md          │
                    │  └── subtitles.vtt          │
                    └─────────────────────────────┘
```

## Компоненты

### 1. QuickRecorder (внешний)

Приложение с иконкой в menu bar, даёт UI для старта/остановки записи и настройки хоткеев. Использует нативный macOS ScreenCaptureKit, поэтому не требует виртуальных аудиодрайверов.

**Почему QuickRecorder, а не OBS?**
OBS мощнее, но перегружен для «нажал и записал». QuickRecorder — минималистичный, открытый, оптимизирован под Apple Silicon.

### 2. fswatch (file system watcher)

Нативная обёртка над `FSEvents API` macOS. Уведомляет скрипт о любых изменениях в `~/Recordings/`. Мы фильтруем только события `Created` и `Renamed` в корне папки (игнорируем подпапки, т.к. в них уже обработанные встречи).

**Почему не polling?**
Polling (проверка каждые N секунд через `while sleep`) жрёт батарею, имеет задержку, и plug-and-play страдает. fswatch работает через ядро macOS и уведомляет моментально.

### 3. auto-transcribe.sh (daemon)

Главный оркестратор. Цикл:

1. Ждёт событие от fswatch
2. Проверяет что файл — аудио/видео по расширению
3. Пауза 3 секунды (дать файлу дописаться на диск)
4. Создаёт папку под встречу, переносит исходник
5. Извлекает аудиодорожку в WAV 16 kHz mono (формат входа Whisper)
6. Запускает whisper-cli
7. Генерирует Markdown с YAML frontmatter
8. Отправляет уведомление через terminal-notifier

Живёт в фоне через **LaunchAgent** (`com.saqta.autotranscribe`) с `KeepAlive` — если упал, перезапустится.

### 4. ffmpeg

Один вызов — конвертация в формат Whisper:

```bash
ffmpeg -i input.mp4 -ar 16000 -ac 1 -c:a pcm_s16le output.wav
```

- `-ar 16000` — 16 kHz (то, что хочет Whisper)
- `-ac 1` — моно (стерео Whisper не нужно)
- `-c:a pcm_s16le` — несжатый PCM для максимальной скорости чтения

### 5. whisper.cpp

C++ порт Whisper с Metal-бэкендом. На M1-M4 запускается через Apple GPU, примерно в 5-10 раз быстрее CPU-версии.

Флаги в Saqta:
- `-m MODEL` — путь к модели
- `-l ru` — язык форсом
- `-f audio.wav` — входной файл
- `--output-txt --output-vtt` — форматы экспорта
- `-of prefix` — префикс для выходных файлов

### 6. terminal-notifier

Mac-native уведомления через Notification Center. Поддерживает `-execute` — выполнить команду при клике по баннеру. Мы используем это чтобы открывать Finder с готовой встречей.

## Почему не Electron/Swift app

- **Приватность** — пользователь видит каждую строку скриптов
- **Прозрачность** — никаких бинарников, всё можно читать и патчить
- **Простота сборки** — не нужен Xcode, signing, notarization
- **Легко менять поведение** — открыл `auto-transcribe.sh`, поправил, сохранил

Минусы:
- Нет graphical UI (пока что)
- Хоткей провайдится QuickRecorder, а не нашим приложением

В [Roadmap](../README.md#️-roadmap) есть план на Swift menu bar app — это даст явную иконку «запись идёт / транскрибируется».

```

├── bin/saqta/                    # Скрипты (копии из репы)
│   ├── auto-transcribe.sh
│   ├── transcribe-file.sh
│   └── launchagents/

    └── com.saqta.autotranscribe.plist
```

## Известные ограничения

- **Один язык на запись** — Whisper лучше всего работает когда язык явно задан. Встреча на смеси рус/англ — см. [`auto` mode](https://github.com/openai/whisper#available-models-and-languages).
- **Нет диаризации** — Whisper сам по себе не различает спикеров. Планируется в v0.2 через раздельные треки QuickRecorder.
- **Последовательная обработка** — если запишешь две встречи подряд до окончания первой транскрибации, вторая будет ждать. Это by design — не перегружать GPU.
