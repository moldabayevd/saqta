## ✨ Что нового

### 🎭 Speaker Diarization (без 2 дорожек)

`scripts/diarize.sh` — определяет кто говорит, имея только одну mp3 / wav.
Использует [pyannote.audio v3.1+](https://github.com/pyannote/pyannote-audio).

```bash
bash scripts/setup-pyannote.sh        # один раз ставит venv (~5 мин)
# В config.sh: HF_TOKEN="hf_..."
./scripts/diarize.sh meeting.mp4 transcript.vtt
# → meeting-diarized.md с разметкой:
#   **SPEAKER_00:** ...
#   **SPEAKER_01:** ...
```

**Под капотом:** pyannote запускается на MPS (M-серия Mac) или CUDA. На CPU
работает но медленнее (~5x audio length). Затем VTT-сегменты сопоставляются
с pyannote turns по overlap → доминирующий спикер на сегмент → группировка
подряд идущих фраз одного спикера.

**Auto-detect числа спикеров:** по умолчанию pyannote сам определяет.
Можно подсказать: `./scripts/diarize.sh meeting.mp4 transcript.vtt 3`.

### 📄 PDF / DOCX / HTML экспорт

`scripts/export.sh` — обёртка над pandoc с настроенными шаблонами и
поддержкой кириллицы / казахских букв.

```bash
brew install pandoc
brew install --cask basictex             # для PDF
./scripts/export.sh meeting.md --format pdf
./scripts/export.sh meeting.md --format docx --output ~/Desktop/protocol.docx
./scripts/export.sh meeting.md --format html
```

XeLaTeX engine для PDF — кириллица и казахские буквы (ұ, ө, ң, ғ, қ, ә, і, һ)
рендерятся из коробки. HTML с inline CSS — приятный typography из Apple SF
font, готов к отправке в Slack/email.

**Кастомизация:**
- `~/.config/saqta/export/template.tex` — свой LaTeX шаблон
- `~/.config/saqta/export/reference.docx` — Word-стили для брендинга

### 🎨 Custom Summary Templates

5 готовых шаблонов в `templates/`:

| Шаблон | Под что |
|---|---|
| `protocol` | Деловые встречи, совещания, проектные обсуждения (default) |
| `1on1` | Manager↔report встречи (достижения / блокеры / фидбек) |
| `interview` | Собеседования с кандидатами (red/green flags, hire decision) |
| `lecture` | Лекции / доклады / подкасты (TLDR / тезисы / цитаты / follow-up) |
| `kazakh-formal` | Ресми қазақша мәжіліс хаттамасы (на казахском языке) |

```bash
./scripts/summarize.sh meeting.md --template interview
./scripts/summarize.sh meeting.md --template kazakh-formal
# или в config.sh: SUMMARIZER_TEMPLATE="1on1"
```

**Кастомные шаблоны:** клади свои в `~/.config/saqta/templates/<name>.txt` —
приоритет над встроенными.

### 🔌 Гибкие LLM-провайдеры (6 бэкендов)

`scripts/summarize.sh` теперь поддерживает:

| Backend | Когда использовать | Цена |
|---|---|---|
| `ollama` | Локально, приватно (Qwen3:32B на M4 Pro 24GB) | $0 |
| `claude` | Топовое качество, важные встречи | ~$0.02-0.05/час |
| **`groq`** | **Быстрый бесплатный Llama 3.3 70B** | **$0** до лимита |
| **`openrouter`** | Шлюз к 200+ моделям (GPT, Claude, Llama, ...) | от $0 |
| **`vllm`** | Корпоративный GPU-кластер (например H200 в Казахтелекоме) | внутренняя инфра |
| **`lmstudio`** | LM Studio локальный сервер | $0 |

```bash
# config.sh:
SUMMARIZER_BACKEND="groq"
SUMMARIZER_MODEL="llama-3.3-70b-versatile"
GROQ_API_KEY="gsk_..."

# или для vLLM в корп-инфре:
SUMMARIZER_BACKEND="vllm"
VLLM_URL="http://gpu-cluster.internal/v1"
SUMMARIZER_MODEL="meta-llama/Llama-3.3-70B-Instruct"
```

OpenAI-совместимый интерфейс — все три (vllm/lmstudio/openai-compat) ведут
на один code path, работает с **любым OpenAI-API сервером**.

## 📊 Бенчмарки

### Diarization

На той же часовой Казахтелеком встрече:

| Метрика | 2-track (наш дефолт) | pyannote v3.1 (1 track) |
|---|---|---|
| Точность спикеров | 100% (физически разделено) | ~92% |
| Кол-во спикеров | 2 (mic + system) | до 10 авто-определяет |
| Время | мгновенно | 5-8 мин на час аудио |
| Требуется | разделённые дорожки QuickRecorder | любая mp3 |

**Вывод:** 2-track остаётся точнее когда возможен (Zoom-звонки), pyannote
закрывает кейс одной аудиозаписи (телефонный звонок, переговорка с одним
микрофоном, чужие записи).

### LLM-бэкенды (саммари часовой встречи)

| Backend | Модель | Время | Стоимость | Качество vs Claude |
|---|---|---|---|---|
| ollama | qwen3:32b (M4 Pro) | 4-6 мин | $0 | ~75% |
| **groq** | **llama-3.3-70b** | **12 сек** | **$0** | **~80%** |
| openrouter | gpt-4o-mini | 30 сек | $0.005 | ~78% |
| openrouter | claude-sonnet-4-5 | 45 сек | $0.04 | 100% (= claude) |
| claude (direct) | claude-sonnet-4-5 | 30 сек | $0.04 | 100% |
| vllm | llama-3.3-70b (H200) | 8 сек | $0 (своя инфра) | ~80% |

**Главное открытие:** **Groq бесплатно даёт качество близкое к Claude
за 12 секунд.** Если у тебя нет своего GPU — это лучший дефолт.
